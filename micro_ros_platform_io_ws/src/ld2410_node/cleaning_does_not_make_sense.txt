Onderstaande sequentie van debug berichten weergeeft eerst een reconnect sequentie.

De esp wacht eerst dat er een verbiding mogelijk is met de agent. Wanneer ik de agent opstart, maakt de esp een verbinding.
Na een verbinding, sluit ik de agent af wacht ik een tijdje en start ik de agent weer op.

Dan de node op de esp wordt gecleant bij een "connection lost" en weer opnieuw geinitaliseerd bij "connection found". Dit is nodig voor een reconnect!

De flow is succes vol echter is de return statemenst van cleaning niet "betrouwbaar". 
Publiser en node zou niet goed opgeschoont zijn, echter lukt het initalizeren vlekkeloos, dit betekent dat het opschonen wel succes vol is gelukt.

Note: Ping agent:1 -> betekent ERROR, Ping agent: 0-> Betekent OK! (Dus verbinding is mogelijk)
Note: Alle waardes in de log waar 0 staat betekent OK, alle andere waardes zijn error waardes.
Volgens de documentatie is 1 -> "Generic error to indicate operation could not complete successfully"
